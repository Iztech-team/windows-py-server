# Baraka POS Printer Server - Windows Edition

Windows port of the Raspberry Pi printer server. No CUPS, no Pillow, no Tailscale.
Uses direct TCP sockets for ESC/POS thermal printing and `adb reverse` for WSA bridge.

## Quick Start

```powershell
# Run the setup script (right-click -> Run with PowerShell, or:)
powershell -ExecutionPolicy Bypass -File .\windows_setup.ps1
```

The setup script will:
1. Check Python 3.8+ is installed
2. Copy files to `%USERPROFILE%\baraka-printer-server`
3. Scan the network for printers on port 9100
4. Set up WSA bridge (`adb reverse`) so the Baraka POS app can reach the server
5. Register a Windows scheduled task for auto-start on login
6. Add a firewall rule for port 3006
7. Start the server

## Manual Start

```powershell
cd %USERPROFILE%\baraka-printer-server
python printer_server.py
```

Or double-click `start_server.bat`.

## API Endpoints

| Method | Endpoint              | Description                         |
|--------|-----------------------|-------------------------------------|
| GET    | `/health`             | Health check + uptime + printer count |
| GET    | `/printers`           | List all configured printers        |
| POST   | `/printers/discover`  | Re-scan network for printers        |
| POST   | `/print`              | Send ESC/POS data (base64)          |
| POST   | `/print/text`         | Print plain text                    |
| POST   | `/print/receipt`      | Print formatted receipt             |
| POST   | `/print/raw`          | Send raw hex bytes to printer       |
| GET    | `/`                   | Server info + all endpoints         |

### Print Text Example

```bash
curl -X POST http://localhost:3006/print/text \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello from Baraka!", "title": "Test Print"}'
```

### Print Receipt Example

```bash
curl -X POST http://localhost:3006/print/receipt \
  -H "Content-Type: application/json" \
  -d '{
    "header": "BARAKA CAFE",
    "subheader": "Order #1234",
    "items": [
      {"name": "Cappuccino", "qty": 2, "price": 4.50},
      {"name": "Croissant", "qty": 1, "price": 3.00}
    ],
    "subtotal": 12.00,
    "tax": 1.80,
    "total": 13.80,
    "footer": "Thank you for visiting!"
  }'
```

### Send Raw ESC/POS (base64)

```bash
curl -X POST http://localhost:3006/print \
  -H "Content-Type: application/json" \
  -d '{"data": "<base64 encoded ESC/POS bytes>", "printer": "printer_1"}'
```

## WSA Bridge

The server automatically sets up `adb reverse` so that the Baraka POS app running
inside Windows Subsystem for Android (WSA) can reach the printer server at
`localhost:3006`. This bypasses WSA's Hyper-V NAT isolation.

**Requirements:**
- WSA installed and running
- WSA Developer Mode enabled (WSA Settings > Developer)
- ADB (Android Platform Tools) installed and in PATH

The bridge auto-reconnects if WSA restarts.

## Architecture

```
┌─────────────────────────────────┐
│  Windows Host                   │
│                                 │
│  ┌──────────────────────┐       │
│  │  Baraka Printer      │       │
│  │  Server (port 3006)  │───────┼──── LAN Printers (port 9100)
│  └──────────┬───────────┘       │     ┌─────────┐
│             │                   │     │Printer 1│
│        adb reverse              │     └─────────┘
│             │                   │     ┌─────────┐
│  ┌──────────▼───────────┐       │     │Printer 2│
│  │  WSA (Android)       │       │     └─────────┘
│  │  Baraka POS App      │       │
│  │  → localhost:3006    │       │
│  └──────────────────────┘       │
└─────────────────────────────────┘
```

## Files

| File                  | Description                                      |
|-----------------------|--------------------------------------------------|
| `printer_server.py`   | Main HTTP server (replaces server1.py)           |
| `printer_discovery.py`| Network scanner for port 9100 printers           |
| `escpos_utils.py`     | ESC/POS command builder (no Pillow)              |
| `wsa_bridge.py`       | ADB reverse tunnel manager for WSA               |
| `windows_setup.ps1`   | One-click setup script                           |
| `start_server.bat`    | Start the server (generated by setup)            |
| `stop_server.bat`     | Stop the server (generated by setup)             |
| `.env`                | Configuration (copy from .env.example)           |
| `printer_registry.json` | Auto-generated printer database                |

## Configuration

Copy `.env.example` to `.env` and edit:

```env
SERVER_PORT=3006
WSA_BRIDGE_ENABLED=true
SCAN_ON_STARTUP=true
```

## Comparison with Raspberry Pi Version

| Feature          | Raspberry Pi          | Windows                    |
|------------------|-----------------------|----------------------------|
| Print system     | CUPS                  | Direct TCP socket          |
| Image printing   | Pillow + python-escpos| Pure ESC/POS text commands |
| Remote access    | Tailscale             | N/A (local only)           |
| WSA support      | N/A                   | adb reverse bridge         |
| Auto-start       | systemd               | Windows Scheduled Task     |
| Printer discovery| nmap + CUPS lpinfo    | Python socket scan         |
| Dependencies     | Many (apt packages)   | Python 3.8+ only           |
